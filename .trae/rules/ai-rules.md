# AI Rules - Frontend Agent Workflow

## 1. 基本原则

- 所有实现必须遵循 docs/ 目录下的文档
- 不允许随意扩展未定义功能
- 优先保证“最小可运行”，而不是功能完整
- 每次只实现一个明确任务

---

## 2. 开发流程约束

在实现代码前，必须：

1. 阅读以下文档：
   - docs/01-prd.md
   - docs/02-technical-design.md
   - docs/03-architecture.md
   - docs/06-todo.md

2. 输出：
   - 本次会修改/新增哪些文件
   - 为什么这么设计

3. 再进行代码实现

---

## 3. 代码生成规则

- 必须写入必要的注释
- 必须使用 TypeScript
- 保持模块解耦
- 禁止跨层直接调用（例如前端直接操作 Agent 层）
- 每个模块职责单一
- 文件命名统一小写 + kebab-case

---

## 4. Agent 设计规则

- 所有 agent 必须是“纯函数式输入输出”
- 不允许在 agent 内部写死逻辑
- 所有 agent 输出必须符合统一 schema
- agent 之间只能通过结构化数据通信

---

## 5. Workflow 规则

- workflow 必须是可扩展的（pipeline / state machine）
- 不允许写死执行顺序
- 每个 step 必须可单独测试

---

## 6. Mock 优先

- 所有功能先用 mock 实现
- 不允许一开始接真实 LLM / 向量库
- mock 数据必须结构化

---

## 7. 输出要求

每次输出必须包含：

1. 修改文件列表
2. 代码实现
3. 如何运行
4. 如何验证

---

## 8. 禁止行为

- 不要一次性生成整个系统
- 不要修改未提及的文件
- 不要引入未说明的依赖
- 不要重构整个项目结构
