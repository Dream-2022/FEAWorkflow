# Frontend Agent Workflow

一个面向前端工程场景的**多 Agent 工作流平台**，支持上传前端项目代码与文档后，自动完成**任务规划、代码/文档检索、影响分析、方案生成、自检审核**，输出结构化改动建议与文档草稿，提升代码库理解和需求改造效率。

## 项目简介

在前端工程开发中，开发者在面对组件增强、需求改造、影响分析、文档补全等任务时，通常需要手动完成阅读代码、检索文件、分析影响、整理文档等流程。这个过程存在以下问题：

- **代码库理解成本高**：对于新项目或复杂项目，上手慢，依赖人工搜读
- **分析链路不可视**：通用聊天式 AI 工具往往只给一段结果，缺少任务拆解、检索、推理、自检过程
- **工程场景适配弱**：单轮问答难以真正结合项目上下文，输出容易“像答案，不像方案”
- **结果不够结构化**：工程任务更需要“涉及文件、影响范围、建议方案、风险点、文档草稿”这类可执行结果

本项目通过构建**多 Agent 工作流平台**，将“工程任务理解 → 上下文检索 → 实现分析 → 方案生成 → 自检审核”形成闭环，并以结构化方式展示结果与过程。

## 核心特性

### ✨ MVP 功能（当前）

| 功能                | 描述                                                  |
| ------------------- | ----------------------------------------------------- |
| **文件上传**        | 支持上传 Markdown、TS/JS、Vue/JSX/TSX 等文件          |
| **自动解析**        | 文件上传后自动解析与切片                              |
| **多 Agent 工作流** | Planner → Retriever → Analyzer → Generator → Reviewer |
| **执行可视化**      | 展示每个 Agent 的状态、输入、输出、耗时               |
| **结构化输出**      | 涉及文件、分析、方案、风险、文档草稿卡片化展示        |
| **历史任务**        | 保存并查看历史任务记录                                |

### 🎯 工作流 Agent

| Agent         | 职责                                       |
| ------------- | ------------------------------------------ |
| **Planner**   | 任务拆解、明确子任务和检索方向             |
| **Retriever** | 从已上传项目中检索最相关上下文             |
| **Analyzer**  | 分析当前实现、依赖关系、影响范围和潜在风险 |
| **Generator** | 生成改动建议、文档草稿、PR 描述等工程结果  |
| **Reviewer**  | 对生成结果做一致性检查和遗漏提示           |

## 快速开始

### 前置要求

- Node.js >= 18
- pnpm 或 npm

### 安装与运行

#### 1. 克隆项目

```bash
git clone <repository-url>
cd FEAWorkflow
```

#### 2. 安装依赖

```bash
pnpm install
```

#### 3. 启动后端服务

```bash
cd server
cp .env.example .env
# 编辑 .env 配置必要的环境变量
pnpm dev
```

后端服务将运行在 `http://localhost:3000`

#### 4. 启动前端服务

打开新的终端：

```bash
cd web
cp .env.example .env
pnpm dev
```

前端服务将运行在 `http://localhost:5173`

## 开发指南

### 开发任务清单

详细的开发任务请查看 [docs/6-todo.md](./docs/6-todo.md)

### 核心流程

一次完整的任务执行链路：

1. **用户上传文件** → 文件被解析并切片存入索引
2. **输入工程任务** → 用户输入需求（如：为 Switch 组件新增 loading）
3. **Planner Agent** → 拆解任务、确定检索方向
4. **Retriever Agent** → 从 RAG 中检索相关代码和文档片段
5. **Analyzer Agent** → 分析当前实现、影响范围、风险点
6. **Generator Agent** → 生成改动建议、文档草稿、PR 描述
7. **Reviewer Agent** → 自检审核、补充遗漏、校准结果
8. **结构化输出** → 前端展示涉及文件、分析、方案、风险、文档
